# Pace

A macOS app developed and shipped without opening the Xcode GUI. The Xcode
project is generated from `project.yml` by [XcodeGen](https://github.com/yonaskolb/XcodeGen);
only the YAML is committed. `*.xcodeproj` is git-ignored and regenerated on demand.

## Layout
- `project.yml` — XcodeGen spec (source of truth for the Xcode project).
- `Sources/PaceCore/` — encrypted store, search, retention, IPC, and shared models.
- `Sources/PaceApp/` — menu-bar UI, hotkey panel, capture, OCR, and paste behavior.
- `Sources/PaceCLI/` — the companion `pace` command-line executable.
- `Tests/PaceCoreTests/` — core store, search, sensitive-content, retention, and IPC tests.
- `App.entitlements` — sandbox/entitlements, generated by XcodeGen from `project.yml`.
- `Configuration/Shared.xcconfig` — committed defaults; `#include?`s the git-ignored
  `Local.xcconfig` which holds your `DEVELOPMENT_TEAM` and `PRODUCT_BUNDLE_IDENTIFIER`.
- `scripts/publish-release.sh` — headless Developer ID archive → notarize → DMG/ZIP → GitHub release.
- `scripts/generate-icons.swift` — draws the app icon + menu bar template icon into
  `Sources/PaceApp/Assets.xcassets` (run `swift scripts/generate-icons.swift` from the repo root).

## Setup (once)
```sh
brew install xcodegen
cp Configuration/Local.xcconfig.example Configuration/Local.xcconfig
# edit Local.xcconfig: set DEVELOPMENT_TEAM and PRODUCT_BUNDLE_IDENTIFIER
xcodegen generate
```

## Regenerate the project (after editing project.yml, or after cloning)
```sh
xcodegen generate
```

## Build & test (headless, unsigned)
```sh
xcodebuild -project Pace.xcodeproj -scheme Pace \
  -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build

xcodebuild -project Pace.xcodeproj -scheme Pace \
  -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO test

xcodebuild -project Pace.xcodeproj -scheme PaceCLI \
  -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build
```

## Run
```sh
xcodebuild -project Pace.xcodeproj -scheme Pace \
  -configuration Debug -derivedDataPath build build
open build/Build/Products/Debug/Pace.app
```

## Release (signed + notarized)
```sh
scripts/publish-release.sh --dry-run     # inspect the release plan
scripts/publish-release.sh               # build a local signed/notarized ZIP + DMG
scripts/publish-release.sh --publish     # + create the GitHub release
```
Set up notarization once with
`scripts/publish-release.sh --setup-notary-profile PaceNotary`, or export
`APPLE_ID`, `APPLE_ID_PASSWORD`, and `TEAM_ID` (see `.env.example`). Secrets never touch git.
The script auto-detects the sole `.xcodeproj` and uses its name as the scheme;
override with `--project` / `--scheme`.

## Conventions
- To change project structure, targets, or build settings, edit `project.yml` and
  run `xcodegen generate` — never edit the `.xcodeproj` by hand.
- Bump `MARKETING_VERSION` / `CURRENT_PROJECT_VERSION` in `project.yml`, regenerate,
  commit, then release. The release script asserts the committed version.
- For a menu-bar/agent app with no Dock icon, set `INFOPLIST_KEY_LSUIElement` to `"YES"`.
